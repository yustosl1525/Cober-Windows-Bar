# v0.8 System Status Native Source Boundary Questions

## Purpose

This document records native source boundary questions for future system status planning. It asks questions only. It does not answer them, define source behavior, or authorize implementation.

This artifact is docs-only and stays at questions, review-boundary, and evidence-gate level. It does not add code, tests, config, package changes, dependencies, Rust, Tauri, runtime behavior, provider behavior, store behavior, resolver behavior, event bus behavior, UI behavior, showcase behavior, native API behavior, source calls, source behavior, real toggles, permissions, capabilities, or `windowsProviders` changes.

The current baseline remains unchanged:

- `windowsProviders: false` remains truthful and unchanged.
- Provider `origin: "native"` with `support: "preflight"` remains diagnostic-only.
- This is native source boundary question planning only, not native/runtime/provider implementation readiness.
- Native system status support is not available, ready, connected, active, implemented, production-capable, or already wired.

## Question Boundary

These questions are for planning review only. They must not become answers in this slice.

The questions must not define:

- Source calls or source behavior.
- APIs, module paths, Rust/Tauri commands, or native API behavior.
- Windows API behavior, native handles, OS observation, polling, or background processes.
- Provider exports, provider lifecycle, provider startup, provider subscription, or real provider work.
- Runtime-provider wiring, runtime payload implementation, event emission, Store behavior, Resolver behavior, UI exposure, or showcase behavior.
- Permissions, capabilities, config toggles, package changes, dependency changes, or implementation mechanics.
